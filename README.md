# Visual-Basis —— GUI Terminal &amp; Frontend Base
Visial Basis，你可以简称VB基座，它是一个为构建大前端而生的底层基座。
目前，大前端的开发适配方向主要集中在对不同OS的GUI API，但是由于各个OS提供的GUI系统有众多差异，难以完全实现大前端。本项目的目的就是在这个问题的基础上，通过开发一套各个OS可公用的GUI基座，然后针对不同设备，可以在这个基座上搭建自己的框架包，不同开发者选取不同的框架开发相应的应用与界面，但同时保持大语言环境不变，从而减少学习成本，最大可能的实现 大前端。

目前的规划：
以web技术为 开发者语言 ，XML（计划放弃HTML5）为界面结构，它构成样式，前端逻辑的操作基准。仍使用CSS、JS作为样式、逻辑语言，但需要做一些小小的改动。
选择原因：随着前端开发经验的累积，HTML、CSS、JS不断经历着实践的检验，我们已经形成了以结构化语言规划结构，样式语言设定样式、逻辑语言复杂操作的基本思维，这个思维在开发上保持了 高内聚，低耦合的开发思想，值得借鉴。但是目前这三个原生语言已经很少使用了，框架，工程化组成了开发的实际使用者，所以我们需要考虑这些新的技术架构，并且支持它们。

我的计划是 参考Broswer的架构实现，桌面、手机端，再开发自己的框架，这样开发者可以在保持语言基础的不变的情况下，只需要关系各个OS的特性进行调整与使用，这是我认为最好，最理想的 大前端实现路径
